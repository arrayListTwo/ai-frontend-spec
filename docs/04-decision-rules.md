# 决策规则

AI 在编码过程中面临多种选择时，应遵循以下决策规则。

## 技术选型决策

| 场景 | 决策 | 说明 |
|------|------|------|
| 状态管理 | Pinia | Vue3 官方推荐 |
| 路由 | Vue Router 4 | Vue3 官方路由 |
| HTTP 请求 | Axios | RuoYi-Vue3 默认 |
| UI 组件库 | Element Plus | 项目技术栈 |
| CSS 方案 | SCSS + BEM | 团队约定 |
| 构建工具 | Vite | Vue3 推荐 |

## 代码模式决策

- 优先使用 Composition API（`<script setup>`）
- 组件命名使用 PascalCase
- 组合式函数使用 `use` 前缀（如 `useTable`）
- 工具函数放入 `src/utils/`
- API 调用放入 `src/api/`

## 冲突解决决策

当规范之间存在冲突时，按以下优先级：

1. 安全性规则（最高优先级）
2. 业务逻辑正确性
3. 项目既有约定
4. 团队编码规范
5. 社区最佳实践（最低优先级）

## 不确定时的处理

当 AI 无法确定最佳方案时：

1. 优先选择最保守、风险最低的方案
2. 向用户说明多种方案的优劣
3. 等待用户决策后再执行
4. 记录决策结果供后续参考

## CR001 使用 Composition API

### 规则编号

CR001

### 目标

统一项目开发模式，保证所有 Vue 页面保持一致的开发风格。

### MUST（必须）

- 使用 Composition API。
- 使用 `<script setup>`。
- 使用 Vue 官方 Composition API（ref、reactive、computed、watch）。
- 新增页面、新增组件均遵循本规则。

### SHOULD（推荐）

- 简单状态优先使用 `ref()`。
- 复杂对象优先使用 `reactive()`。
- 优先使用组合式函数（Composable）封装公共逻辑。

### MUST NOT（禁止）

- 使用 Options API。
- 使用 `export default`。
- 使用 `data()`、`methods`、`computed:{}`、`watch:{}`。
- 使用 `this.xxx`。

### 正确示例

```vue
<script setup>
import { ref } from 'vue'

const loading = ref(false)
</script>
```

### 错误示例

```javascript
export default {
  data() {
    return {
      loading: false
    }
  }
}
```

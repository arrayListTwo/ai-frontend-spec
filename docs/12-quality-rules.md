# 质量规则

本章定义 AI 生成代码的质量标准和检查规则。

## 代码质量标准

### 必须满足

- 代码能正常运行，无语法错误
- 符合项目 TypeScript 配置要求
- 符合 ESLint 规则（如有配置）
- 无硬编码的敏感信息

### 应该满足

- 命名清晰，具有自描述性
- 函数职责单一，长度合理
- 适当的代码注释
- 统一的代码风格

### 建议满足

- 组件可复用
- 工具函数可测试
- 性能合理（无明显性能问题）

## 自检清单

AI 在交付代码前，应完成以下自检：

- [ ] 代码能正常运行
- [ ] 无 TypeScript 类型错误
- [ ] 符合命名规范
- [ ] 导入顺序正确
- [ ] 无冗余代码
- [ ] 无 console.log 残留
- [ ] 样式使用 scoped
- [ ] 权限指令已添加
- [ ] API 调用方式正确
- [ ] 错误处理已完善

## 常见问题

| 问题 | 说明 |
|------|------|
| 遗漏关联修改 | 修改了组件但未更新导入方 |
| 样式污染 | 未使用 scoped 或 BEM |
| 类型不安全 | 使用 any 或缺少类型定义 |
| 硬编码 | 魔法数字或字符串未提取为常量 |
| 权限遗漏 | 未添加 v-hasPermi 指令 |
| 接口不一致 | 前后端字段名不匹配 |

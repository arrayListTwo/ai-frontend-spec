# 禁止规则

本章定义 AI 在编码过程中绝对禁止的行为。

## 禁止操作

| 类别 | 禁止行为 |
|------|----------|
| 安全 | 禁止硬编码密钥、Token、密码等敏感信息 |
| 安全 | 禁止引入未经验证的第三方依赖 |
| 安全 | 禁止修改权限控制相关代码（除非用户明确要求） |
| 数据 | 禁止删除用户数据而不做确认 |
| 数据 | 禁止修改数据库迁移文件（除非用户明确要求） |
| 架构 | 禁止擅自修改全局配置（路由、Store、全局组件） |
| 架构 | 禁止引入新的技术栈或框架（除非用户明确要求） |
| 代码 | 禁止使用 `any` 类型（除非确实无法避免） |
| 代码 | 禁止使用 `var` 声明变量 |
| 代码 | 禁止使用 `console.log` 调试代码（生产代码） |
| 代码 | 禁止使用内联样式（除非动态样式） |
| 规范 | 禁止跳过用户确认执行危险操作 |
| 规范 | 禁止在未理解上下文的情况下编写代码 |
| 规范 | 禁止擅自修改已有代码的业务逻辑 |

## 禁止模式

### 不安全的代码

```typescript
// 禁止
const password = '123456'
const API_KEY = 'sk-xxxx'
localStorage.setItem('token', hardcodedToken)
```

### 不规范的代码

```vue
<!-- 禁止 -->
<div style="color: red; font-size: 16px;">
<!-- 禁止 -->
<div :class="{ active: true }">
```

## 违规处理

如果 AI 发现自己可能违反了禁止规则：

1. 立即停止执行
2. 向用户说明情况
3. 等待用户确认后再继续
4. 记录违规原因供后续改进

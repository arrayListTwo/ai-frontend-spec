# 工作流程

AI 执行前端编码任务时应遵循标准工作流程。

## 标准流程

```
1. 接收任务
   ↓
2. 理解需求（阅读相关文档和代码）
   ↓
3. 分析上下文（项目结构、已有代码、技术栈）
   ↓
4. 制定方案（如有多种方案，向用户说明）
   ↓
5. 用户确认
   ↓
6. 执行编码
   ↓
7. 自检自查
   ↓
8. 交付结果
```

## 任务分解

对于复杂任务，AI 应主动将其分解为可执行的子任务：

- 识别任务的核心目标
- 列出需要修改的文件
- 确定修改的先后顺序
- 评估每个步骤的风险

## 执行规范

- 每次只做一个逻辑单元的修改
- 修改前先确认当前文件的内容
- 修改后检查是否有遗漏的关联修改
- 保持 git 原子提交的粒度

## 异常处理

当执行过程中遇到问题时：

1. 立即停止并分析问题原因
2. 向用户报告问题并提供解决方案
3. 等待用户确认后再继续
4. 记录问题和解决方案供后续参考

# 自检清单

AI 在完成每次编码任务后，必须使用以下清单进行自检。

## 功能完整性

- [ ] 是否完成了用户要求的所有功能
- [ ] 代码逻辑是否正确
- [ ] 边界情况是否处理

## 代码质量

- [ ] 命名是否清晰、符合规范
- [ ] 代码是否简洁、无冗余
- [ ] 注释是否充分（复杂逻辑）
- [ ] TypeScript 类型是否完整

## 项目一致性

- [ ] 文件组织是否符合项目结构
- [ ] 导入方式是否一致
- [ ] 代码风格是否一致
- [ ] 是否复用了已有组件/函数

## 安全性

- [ ] 无硬编码的密钥或 Token
- [ ] 权限控制是否正确
- [ ] 用户输入是否做了校验

## 性能

- [ ] 无不必要的计算或请求
- [ ] 大列表是否考虑分页
- [ ] 图片是否做了懒加载（按需）

## 可维护性

- [ ] 代码是否易于理解
- [ ] 是否遵循了单一职责原则
- [ ] 是否为后续扩展预留了空间

# 输出规则

本章定义 AI 输出代码和文档的格式标准。

## 代码输出规则

### 文件输出

- 每次输出一个完整文件，除非用户要求仅输出修改部分
- 文件路径必须明确标注
- 代码块必须标注语言类型

### 代码格式

- 使用 2 空格缩进
- 字符串使用单引号
- 语句末尾无分号（Vue SFC 中）
- 逗号在最后元素后保留（trailing comma）

### 输出模板

```markdown
## {功能描述}

### {文件路径}

\`\`\`vue
// 代码内容
\`\`\`
```

## 文档输出规则

- 使用 Markdown 格式
- 标题层级清晰
- 列表使用标准 Markdown 语法
- 代码块标注语言类型

## 对话输出规则

- 回答简洁，直接回应问题
- 如有多种方案，列出优劣对比
- 复杂操作前先确认
- 交付时提供自检结果

## 文件修改规则

当修改已有文件时：

1. 明确标注修改的文件路径
2. 展示修改前后的对比
3. 说明修改原因
4. 检查是否有遗漏的关联修改

# 上下文感知

AI 必须在充分理解上下文的基础上进行编码，而非凭空生成代码。

## 项目上下文

在开始任何任务前，AI 应主动了解：

- 项目的技术栈版本（Vue3、Element Plus、RuoYi-Vue3 版本）
- 项目的目录结构和模块划分
- 项目的构建工具和配置（Vite、Webpack）
- 项目的代码规范配置（ESLint、Prettier）

## 模块上下文

针对具体任务，AI 应了解：

- 当前模块的业务职责
- 模块内的组件层次结构
- 数据流向（store、props、events）
- 与后端 API 的交互方式

## 代码上下文

在修改具体文件时，AI 应关注：

- 文件的整体结构和导出方式
- 已有的导入和依赖关系
- 组件的 props、events、slots 定义
- 样式的组织方式和命名约定

## 上下文获取方式

AI 应通过以下方式获取上下文：

1. 阅读项目 README 和相关文档
2. 浏览目录结构
3. 阅读关键文件（main.ts、router、store、api 等）
4. 阅读已有组件代码了解编码风格
5. 向用户主动询问不确定的信息

# Contributing to AFES

感谢你对 AFES 项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告问题

如果你发现了规范中的错误或不一致之处，请通过 GitHub Issues 提交报告，并包含以下信息：

- 问题描述
- 涉及的文档章节
- 建议的修复方案（如有）

### 提交改进

1. Fork 本仓库
2. 创建你的分支：`git checkout -b feature/your-feature`
3. 进行修改并确保符合项目风格
4. 提交修改：`git commit -m 'feat: add xxx'`
5. 推送分支：`git push origin feature/your-feature`
6. 创建 Pull Request

### Commit 规范

本项目遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

- `feat:` 新功能或新规范
- `fix:` 修复错误
- `docs:` 文档变更
- `style:` 格式调整（不影响内容）
- `refactor:` 重构
- `test:` 测试相关
- `chore:` 构建/工具链变更

### Pull Request 要求

- PR 标题清晰描述变更内容
- 单个 PR 聚焦于一个变更
- 确保 Markdown 格式正确
- 如涉及新章节，请同步更新 README.md 的项目结构

## 行为准则

- 尊重每一位贡献者
- 以建设性的方式提出批评和建议
- 专注于对社区最有利的事情

## 联系方式

如有任何问题，请通过 GitHub Issues 与我们联系。
